Qu'est-ce que CompactVD ? ▲

CompactVD est un utilitaire pour optimiser la taille des images de disque dynamique, basé sur les tables d'allocation bitmap des systèmes de fichiers connus.
Vous pouvez vérifier s'il y a de l'espace perdu dans une image disque dynamique en comparant la taille du fichier image disque avec l'espace utilisé par les systèmes de fichiers à l'intérieur de l'image. Pour une allocation optimale des ressources, ils ne doivent pas être très différents. Ou faites simplement glisser une image disque dans cette fenêtre :
Les systèmes de fichiers actuellement pris en charge sont :
  • Système de fichiers de nouvelle technologie (NTFS), pour les ordinateurs Windows
  • Système de fichiers hiérarchique ou Apple (HFS+ ou APFS), pour macOS
  • Système de fichiers étendu (EXT), utilisé dans la plupart des bureaux Linux
Fait aussi la lecture du BTRFS ou XFS (Linux), mais dans ces cas, la compression ne fonctionne pas.

Image de disque compact en place ▲

L'opération de compactage extrait des blocs de données de la fin de l'image disque vers l'espace qui a été détecté comme n'étant pas utilisé par les systèmes de fichiers.
Pour commencer, une analyse est effectuée pour détecter les blocs de données qui ne sont pas utilisés par les systèmes de fichiers, et la structure de l'image disque est mise à jour en marquant ces blocs comme s'ils n'avaient jamais été utilisés.
Ensuite, les blocs avec des données effectives sont déplacés de la fin de l'image disque vers l'espace qui a été récupéré par l'opération précédente.
Enfin, la structure de l'image est mise à jour pour refléter la nouvelle position des blocs déplacés et la taille du fichier image est réduite. Les mises à jour de la structure de l'image sont temporairement enregistrées sur le support hôte et apparaitront sur l'image lors de la prochaine exécution du programme, en cas de panne matérielle récupérable.

Copier sur une nouvelle image disque ▲

L'opération de copie commence par une analyse pour détecter les blocs de données qui ne sont pas utilisés par les systèmes de fichiers. La nouvelle image est créée en ignorant ces blocs et les blocs qui seraient complètement remplis de zéros.
Le format de la nouvelle image disque peut être sélectionné dans la boîte de dialogue Copier l'image disque avec l'une de ces extensions de fichier :
  • VDI, Virtual Disk Image pour Oracle VirtualBox
  • VMDK, Virtual Machine Disk pour les produits VMware
  • VHD, Virtual Hard Disk pour les produits Microsoft
  • RAW, pour une image disque complète
