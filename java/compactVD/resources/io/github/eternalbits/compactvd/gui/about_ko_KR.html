CompactVD란 무엇입니까? ▲

CompactVD는 알려진 파일 시스템의 비트맵 할당 테이블을 기반으로 동적 디스크 이미지의 크기를 최적화하는 유틸리티입니다.
디스크 이미지 파일의 크기와 이미지 내부의 파일 시스템이 사용하는 공간을 비교하여 동적 디스크 이미지에 낭비되는 공간이 있는지 확인할 수 있습니다. 최적의 리소스 할당을 위해 크게 다르지 않아야 합니다. 또는 디스크 이미지를 이 창으로 드래그하십시오.
현재 지원되는 파일 시스템은 다음과 같습니다.
  • NTFS(New Technology File System), Windows 컴퓨터용
  • macOS용 계층적 또는 Apple 파일 시스템(HFS+ 또는 APFS)
  • 대부분의 Linux 데스크톱에서 사용되는 확장 파일 시스템(EXT)
또한 BTRFS 또는 XFS(Linux)를 읽지만 이러한 경우 압축이 작동하지 않습니다.

제자리에 컴팩트 디스크 이미지 ▲

압축 작업은 디스크 이미지의 끝에서 파일 시스템에서 사용하지 않는 것으로 감지된 공간으로 데이터 블록을 가져옵니다.
스캔을 시작하여 파일 시스템에서 사용하지 않는 데이터 블록을 감지하고 디스크 이미지 구조를 업데이트하여 해당 블록을 사용하지 않은 것처럼 표시합니다.
그런 다음 유효 데이터가 있는 블록이 디스크 이미지 끝에서 이전 작업으로 복구된 공간으로 이동됩니다.
마지막으로 이동된 블록의 새 위치를 반영하도록 이미지 구조가 업데이트되고 이미지 파일 크기가 잘립니다. 이미지 구조에 대한 업데이트는 호스트 미디어에 일시적으로 저장되며 복구 가능한 하드웨어 오류가 발생한 경우 다음 프로그램 실행 시 이미지에 기록됩니다.

새 디스크 이미지로 복사 ▲

복사 작업은 파일 시스템에서 사용하지 않는 데이터 블록을 검색하는 스캔으로 시작됩니다. 새 이미지는 해당 블록과 그렇지 않으면 완전히 0으로 채워지는 블록을 무시하고 생성됩니다.
새 디스크 이미지의 형식은 다음 파일 확장자를 사용하여 디스크 이미지 복사 대화 상자에서 선택할 수 있습니다.
  • VDI, Oracle VirtualBox용 가상 디스크 이미지
  • VMDK, VMware 제품용 가상 머신 디스크
  • Microsoft 제품용 VHD, 가상 하드 디스크
  • RAW, 완전한 디스크 이미지용
